单服推送信息到中央服务器（同种服务器代码，通过标志位，区分）
1.保持原有的逻辑（同时保证兼容）
2.修改（单服，请求不响应，单服将请求转发给中央服务器，中央服务器将结果，反馈给单服，单服再响应给客户端）
3.本质是
玩家唯一标志
	编码：
	玩家标志 + 单服标志 = 中央服务器玩家的标志
	client_flag + single_server_flag => center_server_flag
	解码：
	center_server_flag => client_flag + single_server_flag
过程
	玩家（请求） -》 单服 （玩家信息）-》中央服务器（战斗逻辑）-》单服（响应） -》玩家
